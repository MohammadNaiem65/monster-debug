3. Describe Event loop in JavaScript.

JavaScript has a runtime model based on an Event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks. This model is quite different from models in other languages like C and Java.

JavaScript executes it's codes in three parts. 1.Heap, 2.Queue, 3.Event loop.

Firstly, Objects are allocated in a heap which is just a name to denote a large (mostly unstructured) region of memory. Then, A JavaScript runtime uses a message queue, which is a list of messages to be processed. Each message has an associated function that gets called to handle the message. Now the "Event Loop's" comes in. queue.waitForMessage() waits synchronously for a message to arrive (if one is not already available and waiting to be handled).

4. How do we get Undefined in our code.

undefined is a property of the global object. That is, it is a variable in global scope. The initial value of undefined is the primitive value undefined. Usually, we get undefined in 6 cases.

1. If we initialize a variable but don't define it's value, it will give undefined.

For example:
let i;
console.log(i);

2. Function that returns nothing will give undefined to a variable, the value of that variable will be undefined.

For example:
const multiply = (a, b) => {
const result = a * b;
return;
}
console.log(add(2, 4));

3. Function that does not contain return statement, the value of that will be undefined.

For example:
const add = (a, b) => {
const result = a + b;
}
console.log(add(2, 4));

4. If enough arguments will not pass to a function, it will give undefined.

For example:
const add = (a, b) => {
const result = a + b;
}
console.log(add(2));

5. If we console.log a property that does not exist in an object, it will give undefined.

For example:
const myself = { name: "Naiem", age: 18, jod: "learning" };
console.log(myself.salary)

6. If we console.log a value from an array with index of which index value does not exist, it will give undefined.

For example:
const marks = [24, 54, 12, 53, 23];
console.log(marks[10])

Besides this there are many others cases for which we can get undefined. For instance, if we delete a value from an array an try to console.log it, it will give undefined.

For example:
const marks = [24, 54, 12, 53, 23];
marks.pop()
console.log(marks[4])
